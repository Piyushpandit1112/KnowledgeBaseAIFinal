Manufacture

The API offers the ability to list, create, edit and delete manufacture.

List manufactures - GET /v1/manufactures

Listing the manufactures returns, depending on the parameters provided, a list of manufactures including business information, address information and risk information.

The request additionally supports the pagination parameters for index listings.

Response

The response is an array of manufacture objects. It additionally includes the pagination parameters for index listings.

Attribute Format Description
manufactures Array Array of all requested manufactures. See list a manufacture for content details.
NOTE: One manufacture in the array does not have the manufacture namespace as stated in the 'list a manufacture' response.

Examples

Return all manufactures available in a paginated manner.

Execute
GET /v1/manufactures.json

List a manufacture - GET /v1/manufactures/:id

Returns one single manufacture based on the provided id.

Response

Attribute Format Description
The following objects live in the manufacture namespace.
id Number The internal manufacture id.
name String The manufacture name.
business_unit_ids Array An array of all assigned business unit ids.
material_group_ids Array An array of all assigned material group ids.
responsible_user_ids Array An array of all assigned responsible user ids.
The following objects live in the manufacture.address namespace.
address1 String The street including house number.
address2 String Additional information on the address.
zip_code String The city's zip code.
city String The city name.
state String The state name.
country_code String The country's code in ISO 3166-1 alpha 2 format.
latitude Number Geographic latitude coordinate in the signed degrees format.
longitude Number Geographic longitude coordinate in the signed degrees format.
The following objects live in the manufacture.score_card namespace.
If manufacture.score_card is null, it means that this manufacture is currently not being monitored.
The score card displayed depends on the configured master score card.
risk_score Number The risk score on a scale from 0 to 100. null indicates non-availablity.
risk_groups Array An array of all risk_groups inside this score_card.
The following objects are one element in the manufacture.score_card.risk_groups namespace.
name String The risk group name.
risk_score Number The risk group's risk score on a scale from 0 to 100. null indicates non-availablity.
risks Array An array of all risks inside one risk_group.
The following objects are one element in the manufacture.score_card.risk_groups.risks namespace.
name String The risk name.
risk_score Number The risk's risk score on a scale from 0 to 100. null indicates non-availablity.
indicators Array An array of all indicators inside one risk.
The following objects are one element in the manufacture.score_card.risk_groups.risks.indicators namespace.
name String The indicator name.
risk_score Number The indicator's risk score on a scale from 0 to 100. null indicates non-availablity.

Example

Return the manufactures with id 322038.

Execute
GET /v1/manufactures/322038.json

Create a manufacture - POST /v1/manufactures

Creates a manufacture with the given attributes.

Parameter Type Format Description
The following data needs to be inside the manufacture namespace.
name Required String The name of the new manufacture.
duns Optional String The DUNS has to be in the format XX-XXX-XXXX.
business_unit_ids Optional Array Array of existing business unit ids that should be assigned to the created manufacture.
material_group_ids Optional Array Array of existing material group ids that should be assigned to the created manufacture.
responsible_user_ids Optional Array Array of existing responsible user ids that should be added to the manufacture.
The following data needs to be inside the manufacture.address namespace.
address1 Required String The street including house number.
address2 Optional String Additional information on the address.
zip_code Required String The city's zip code.
city Required String The city name.
state Optional String The state name.
country_code Required String The country's code in ISO 3166-1 alpha 2 format.

Response

See response codes for what codes will be responded with.

Example

POST /v1/manufactures.json 
  {
    "manufacture": {
      "name": "New Manufacture",
      "business_unit_ids": [6643],
      "material_group_ids": [21619],
      "responsible_user_ids": [%= data.user_id %>],
      "address": {
        "address1": "Orleansstraße 4",
        "city": "München",
        "zip_code": "81669",
        "country_code": "de"
      }
    }
  }

Update a manufacture - PUT /v1/manufactures/:id

Updates the manufacture. The following information can be updated:

Attribute Format Description
The following data needs to be inside manufacture namespace.
name String The manufacture name.
duns string The manufacture's DUNS in the format XX-XXX-XXXX.
business_unit_ids Array Array of existing business unit ids that should be added to the manufacture.
material_group_ids Array Array of existing material group ids that should be added to the manufacture.
responsible_user_ids Array Array of existing responsible user ids that should be added to the manufacture.
The following data needs to be inside manufacture.address namespace.
address1 String The street including house number.
address2 String Additional information on the address.
zip_code String The city's zip code.
city String The city name.
state String The state name.
country_code String The country's code in ISO 3166-1 alpha 2 format.

Response

See response codes for what codes will be responded with.

Example

Update the name of a manufacture.

PUT /v1/manufactures/322038.json 
  {
    "manufacture": {
      "name": "New manufacture name"
    }
  }
